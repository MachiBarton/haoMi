# 豪密密码加密工具 - 需求文档

## 1. 项目概述

### 1.1 项目背景
豪密是中国共产党早期使用的一套高级密码通信系统，由周恩来（化名"伍豪"）于1930年代亲自编制。本项目旨在通过现代Web技术重现这一历史密码系统，让用户能够体验和学习这段红色密码史。

### 1.2 项目目标
- 实现一个可运行的豪密加密/解密工具
- 采用纯前端架构，无需后端服务
- 提供直观的历史密码学习体验
- 支持密码本的导入导出和自定义

### 1.3 用户群体
- 密码学爱好者
- 历史研究者
- 红色文化学习者

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 加密功能
- 输入：明文（中文字符串）
- 输出：密文（数字组，用连字符分隔）
- 过程可视化：显示每个汉字的明码和乱数偏移

#### 2.1.2 解密功能
- 输入：密文（数字组）
- 输出：明文（中文字符串）
- 错误处理：无效密文提示

#### 2.1.3 密码本管理
- 明码本查看（分页表格，300字/页）
- 密码本导入（JSON格式）
- 密码本导出（JSON格式）
- 乱数表查看（每页乱数序列）

### 2.2 辅助功能
- 一键复制密文
- 加密统计信息（字数、耗时）
- 本地存储密码本

## 3. 技术方案

### 3.1 加密算法（页码+位置+乱数三层加密）

**明码编码规则：**
- 每个汉字编码 = 页码(2位) + 行号(2位) + 列号(2位) = 6位数字
- 密码本分页：每页300字（10行 × 30列）
- 示例："中"在第05页第03行第12列 → 明码 = 050312

**乱数生成规则：**
- 每页有一个随机种子（randomSeed）
- 乱数 = (seed × pageNumber × position) % 1000000
- 确保收发双方使用相同密码本即可同步

**加密公式：**
```
密文 = (明码 + 乱数) % 1000000
```

**解密公式：**
```
明码 = (密文 - 乱数 + 1000000) % 1000000
```

### 3.2 密码本数据结构

```typescript
interface CodebookPage {
  pageNumber: number;      // 页码 (1-99)
  rows: number;            // 行数 (固定10)
  cols: number;            // 列数 (固定30)
  characters: string[];    // 300个汉字数组
  randomSeed: number;      // 本页乱数种子
}

interface Codebook {
  name: string;            // 密码本名称
  version: string;         // 版本号
  totalPages: number;      // 总页数
  pages: CodebookPage[];   // 页面数组
}
```

### 3.3 默认密码本
- 使用《千字文》前3000字作为基础字库
- 共10页，每页300字
- 每页分配不同的随机种子

## 4. 界面设计

### 4.1 整体布局
- 左右分栏：左侧40%（密码本区域），右侧60%（操作区域）
- 复古红色特工风格
- 配色：深红色(#8a0000) + 米黄色(#fcf8f3)

### 4.2 左侧区域（密码本）
- 标签页切换：明码本 / 密文表 / 乱数表
- 明码本：10×30网格表格，hover显示坐标
- 分页器：上一页 / 页码输入 / 下一页
- 导入/导出按钮

### 4.3 右侧区域（操作）
- 标签页切换：🔒加密 / 🔓解密
- 明文输入框（textarea）
- 加密过程可视化（显示明码、乱数、密文）
- 执行按钮
- 密文输出框（只读，可复制）

### 4.4 Header
- 标题：豪密 (Hao Mi)
- 副标题：伍豪编制 · 1930
- 安全连接状态显示

### 4.5 Footer
- 当前密码本名称
- 连接状态
- 字数统计、用时统计
- "机密"标识

## 5. 非功能需求

### 5.1 性能要求
- 加密/解密响应时间 < 100ms（100字以内）
- 页面加载时间 < 2s
- 支持最多1000字同时加密

### 5.2 兼容性
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### 5.3 安全要求
- 纯前端实现，不传输任何数据到服务器
- 密码本存储在localStorage
- 敏感操作需要确认
