# 豪密密码加密工具 - 产品需求文档 (PRD)

## 1. 项目背景

### 1.1 历史背景
豪密是中国共产党早期使用的一套高级密码通信系统，由周恩来（化名"伍豪"）于1930年代亲自编制。这套密码系统在革命战争年代发挥了重要作用，是红色密码史上的经典之作。

### 1.2 项目愿景
通过现代Web技术重现这一历史密码系统，让用户能够：
- 体验和学习这段红色密码史
- 理解传统加密算法的原理
- 获得独特的历史沉浸式体验

### 1.3 目标用户
| 用户类型 | 特征 | 需求 |
|---------|------|------|
| 密码学爱好者 | 对古典密码感兴趣 | 了解算法原理，体验加密过程 |
| 历史研究者 | 研究红色历史 | 学习豪密的历史背景和使用方式 |
| 红色文化学习者 | 学习党史 | 通过互动方式了解革命历史 |
| 普通用户 | 好奇尝鲜 | 简单易用的加密工具 |

## 2. 功能清单

### 2.1 核心功能

#### 2.1.1 加密功能 (P0)
- **功能描述**: 将明文（中文/数字/英文）转换为数字密文
- **输入**: 字符串（中文、数字、英文字母，最多1000字）
- **输出**: 7位数字组，用连字符分隔（如：7123456-8123456-9234567-...）
- **加密规则**:
  - 中文：7位数字（0-6或9开头），使用密码本加乱数加密
  - 数字：7位数字（7开头），格式为 7 + 1位数字 + 5位随机
  - 英文：7位数字（8开头），格式为 8 + 3位ASCII码 + 3位随机
- **标点处理**: 自动过滤标点符号，不参与加密
- **错误处理**: 字符不在密码本中时给出提示

#### 2.1.2 解密功能 (P0)
- **功能描述**: 将数字密文还原为明文
- **输入**: 7位数字组密文（支持中文、数字、英文混合）
- **输出**: 原始字符串
- **错误处理**:
  - 无效密文格式提示
  - 无法解密时给出明确错误信息

#### 2.1.3 密码本管理 (P0)
- **明码本查看**:
  - 10×30网格表格展示
  - 分页显示（300字/页）
  - Hover显示坐标位置
- **乱数表查看**: 显示每页的乱数序列
- **密码本导入**: 支持txt格式文件导入（提取汉字生成密码本）
- **密码本导出**: 导出当前密码本为txt文件
- **默认密码本**: 从《三国演义》文本加载

### 2.2 辅助功能

#### 2.2.1 文本操作 (P1)
- 一键复制密文到剪贴板
- 明文/密文清空功能
- 加密/解密模式独立输入状态

#### 2.2.2 统计信息 (P1)
- 字数统计
- 加密/解密耗时显示
- 当前密码本信息

#### 2.2.3 摩尔斯电码发报 (P1)
- 将密文转换为摩尔斯电码
- 自动播放发报音效
- 手动按键发报（J键=点，K键=划）
- 逐字符移动可视化

## 3. 用户故事

### 3.1 加密场景

**故事1：基础加密**
> 作为密码学爱好者，我想要输入一段中文文本并获得对应的密文，以便了解豪密加密的效果。

**验收标准：**
- 可以在明文输入框输入中文字符
- 点击"执行加密"后显示密文
- 密文格式为7位数字组，用连字符分隔

**故事2：多层加密体验**
> 作为学习者，我想要看到加密支持中文、数字、英文的混合加密。

**验收标准：**
- 支持中文加密（使用密码本）
- 支持数字加密（7开头）
- 支持英文加密（8开头）
- 标点符号自动过滤

**故事3：处理不存在字符**
> 作为用户，当输入的中文字符不在密码本中时，我希望得到明确的提示。

**验收标准：**
- 检测不在密码本中的汉字
- 显示错误提示，指出具体字符

### 3.2 解密场景

**故事4：基础解密**
> 作为接收方，我想要输入密文并还原出原始明文，以便阅读加密消息。

**验收标准：**
- 可以在密文输入框输入数字密文
- 点击"执行解密"后显示明文
- 明文与原始加密内容一致

**故事5：错误密文处理**
> 作为用户，当输入无效密文时，我希望得到清晰的错误提示。

**验收标准：**
- 检测密文格式错误
- 显示友好的错误提示

### 3.3 密码本管理场景

**故事6：查看密码本**
> 作为学习者，我想要查看密码本的结构和内容，以便理解编码规则。

**验收标准：**
- 以10×30网格显示密码本
- 支持分页浏览
- Hover显示字符坐标

**故事7：导入自定义密码本**
> 作为高级用户，我想要导入自定义的文本文件作为密码本。

**验收标准：**
- 支持上传txt格式文本文件
- 自动提取汉字生成密码本
- 按300字/页自动分页

**故事8：导出密码本**
> 作为用户，我想要导出当前密码本，以便在其他设备上使用或备份。

**验收标准：**
- 一键导出为txt文件
- 文件包含完整的密码本汉字

### 3.4 摩尔斯电码场景

**故事9：自动发报**
> 作为用户，我想要听到密文的摩尔斯电码发报声音，体验历史发报场景。

**验收标准：**
- 点击"模拟发报"按钮打开发报界面
- 自动播放摩尔斯电码声音
- 可视化显示当前发送位置

**故事10：手动发报**
> 作为用户，我想要手动模拟发报操作，体验发报员的工作。

**验收标准：**
- 支持J键发送"点"，K键发送"划"
- 按键时有声音反馈
- 实时显示发报进度

## 5. 数据结构

### 5.1 密码本结构

```typescript
interface CodebookPage {
  pageNumber: number;      // 页码 (1-999)
  rows: number;            // 行数 (固定10)
  cols: number;            // 列数 (固定30)
  characters: string[];    // 300个汉字数组
  randomSeed: number;      // 本页乱数种子
}

interface Codebook {
  name: string;            // 密码本名称
  version: string;         // 版本号
  totalPages: number;      // 总页数
  pages: CodebookPage[];   // 页面数组
}
```

### 5.2 加密结果结构

```typescript
interface EncryptionResult {
  plaintext: string;       // 原文
  ciphertext: string;      // 密文（连字符分隔的7位数字）
  details: {
    char: string;          // 字符
    type: CharType;        // 类型：chinese/digit/letter
    clearCode: string;     // 明码（中文）
    random: number;        // 乱数
    cipherCode: string;    // 密文（7位）
  }[];
  duration: number;        // 耗时(ms)
}
```

### 5.3 明码编码规则

**中文明码**：7位数字
- 页码（3位）+ 行号（2位）+ 列号（2位）
- 示例：第05页第03行第12列 → 明码 = 0050312

**密文格式**：
- 中文：7位数字（0-6或9开头）
- 数字：7位数字（7开头）
- 英文：7位数字（8开头）
