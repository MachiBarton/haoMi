# 豪密密码加密工具 - 产品需求文档 (PRD)

## 1. 项目背景

### 1.1 历史背景
豪密是中国共产党早期使用的一套高级密码通信系统，由周恩来（化名"伍豪"）于1930年代亲自编制。这套密码系统在革命战争年代发挥了重要作用，是红色密码史上的经典之作。

### 1.2 项目愿景
通过现代Web技术重现这一历史密码系统，让用户能够：
- 体验和学习这段红色密码史
- 理解传统加密算法的原理
- 获得独特的历史沉浸式体验

### 1.3 目标用户
| 用户类型 | 特征 | 需求 |
|---------|------|------|
| 密码学爱好者 | 对古典密码感兴趣 | 了解算法原理，体验加密过程 |
| 历史研究者 | 研究红色历史 | 学习豪密的历史背景和使用方式 |
| 红色文化学习者 | 学习党史 | 通过互动方式了解革命历史 |
| 普通用户 | 好奇尝鲜 | 简单易用的加密工具 |

## 2. 功能清单

### 2.1 核心功能

#### 2.1.1 加密功能 (P0)
- **功能描述**: 将中文明文转换为数字密文
- **输入**: 中文字符串（最多1000字）
- **输出**: 6位数字组，用连字符分隔（如：248135-928102-...）
- **加密过程可视化**: 显示每个汉字的明码、乱数偏移、最终密文
- **错误处理**: 字符不在密码本中时给出提示

#### 2.1.2 解密功能 (P0)
- **功能描述**: 将数字密文还原为中文明文
- **输入**: 6位数字组密文
- **输出**: 中文字符串
- **错误处理**:
  - 无效密文格式提示
  - 无法解密时给出明确错误信息

#### 2.1.3 密码本管理 (P0)
- **明码本查看**:
  - 10×30网格表格展示
  - 分页显示（300字/页）
  - Hover显示坐标位置
- **密文表查看**: 显示明码到密文的映射关系
- **乱数表查看**: 显示每页的乱数序列
- **密码本导入**: 支持JSON格式文件导入
- **密码本导出**: 导出当前密码本为JSON文件

### 2.2 辅助功能

#### 2.2.1 文本操作 (P1)
- 一键复制密文到剪贴板
- 明文/密文清空功能
- 示例文本一键填充

#### 2.2.2 统计信息 (P1)
- 字数统计
- 加密/解密耗时显示
- 当前密码本信息

#### 2.2.3 数据持久化 (P1)
- 密码本本地存储（localStorage）
- 自动保存草稿
- 历史记录（最近10次操作）

## 3. 用户故事

### 3.1 加密场景

**故事1：基础加密**
> 作为密码学爱好者，我想要输入一段中文文本并获得对应的密文，以便了解豪密加密的效果。

**验收标准：**
- 可以在明文输入框输入中文字符
- 点击"执行加密"后显示密文
- 密文格式为6位数字组，用连字符分隔

**故事2：加密过程学习**
> 作为学习者，我想要看到每个汉字的加密过程细节，以便理解豪密的加密原理。

**验收标准：**
- 显示每个汉字的明码（页码+行号+列号）
- 显示乱数偏移量
- 显示最终密文
- 过程可视化展示

**故事3：处理不存在字符**
> 作为用户，当输入的字符不在密码本中时，我希望得到明确的提示。

**验收标准：**
- 检测不在密码本中的字符
- 显示错误提示，指出具体字符
- 跳过该字符继续加密其他字符

### 3.2 解密场景

**故事4：基础解密**
> 作为接收方，我想要输入密文并还原出原始明文，以便阅读加密消息。

**验收标准：**
- 可以在密文输入框输入数字密文
- 点击"执行解密"后显示明文
- 明文与原始加密内容一致

**故事5：错误密文处理**
> 作为用户，当输入无效密文时，我希望得到清晰的错误提示。

**验收标准：**
- 检测密文格式错误
- 显示友好的错误提示
- 提供格式修正建议

### 3.3 密码本管理场景

**故事6：查看密码本**
> 作为学习者，我想要查看密码本的结构和内容，以便理解编码规则。

**验收标准：**
- 以10×30网格显示密码本
- 支持分页浏览
- Hover显示字符坐标

**故事7：导入自定义密码本**
> 作为高级用户，我想要导入自定义的密码本，以便使用个性化的加密方案。

**验收标准：**
- 支持上传JSON格式密码本文件
- 验证密码本格式正确性
- 导入成功后自动切换

**故事8：导出密码本**
> 作为用户，我想要导出当前密码本，以便在其他设备上使用或备份。

**验收标准：**
- 一键导出为JSON文件
- 文件包含完整的密码本数据
- 导出文件名包含版本信息

## 7. 数据结构

### 7.1 密码本结构

```typescript
interface CodebookPage {
  pageNumber: number;      // 页码 (1-99)
  rows: number;            // 行数 (固定10)
  cols: number;            // 列数 (固定30)
  characters: string[];    // 300个汉字数组
  randomSeed: number;      // 本页乱数种子
}

interface Codebook {
  name: string;            // 密码本名称
  version: string;         // 版本号
  totalPages: number;      // 总页数
  pages: CodebookPage[];   // 页面数组
}
```

### 7.2 加密结果结构

```typescript
interface EncryptionResult {
  plaintext: string;       // 原文
  ciphertext: string;      // 密文（连字符分隔）
  details: {
    char: string;          // 汉字
    clearCode: string;     // 明码（6位）
    random: number;        // 乱数
    cipherCode: string;    // 密文（6位）
  }[];
  duration: number;        // 耗时(ms)
}
```

## 8. 错误处理

### 8.1 错误类型

| 错误码 | 描述 | 用户提示 |
|--------|------|---------|
| CHAR_NOT_IN_CODEBOOK | 字符不在密码本 | "字符[X]不在当前密码本中，已跳过" |
| INVALID_CIPHER_TEXT | 密文格式错误 | "密文格式错误，应为6位数字组" |
| INVALID_CODEBOOK | 密码本格式错误 | "密码本格式不正确，请检查JSON文件" |
| STORAGE_ERROR | 存储错误 | "本地存储失败，请检查浏览器设置" |

### 8.2 错误展示
- 使用红色Alert组件
- 显示在操作区域顶部
- 3秒后自动消失
